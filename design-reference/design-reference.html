Scaling micro-frontends with Module Federation

2026-06-28·9 min read·2,140 words·ARCHITECTURE

Runtime composition changes the economics of a large frontend. Instead of one deploy pipeline gating every team, each remote ships on its own cadence and the shell resolves them at load. The trade you accept is a versioning contract at the boundary — and that contract is where most of the real work lives.

next.config.tscopy
// shared singletons keep one React on the page
new ModuleFederationPlugin({
  name: "shell",
  remotes: { checkout: "checkout@/mf.js" },
  shared: { react: { singleton: true } }
})
Treat every remote's exposed surface as a public API. If it isn't typed, it isn't a contract.
