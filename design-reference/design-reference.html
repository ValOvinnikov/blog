Atom

Text

Serif body copy atom. Ships 4 variants — a superset of the default/muted tones shown below, each pairing a font-size token with a color and line-height:

  • lead (default) — text-lead (0.9375rem) / leading-[1.72] / text-text. Standard long-form body copy.
  • muted — same size/line-height as lead, text-muted color. The de-emphasized tone (bios, secondary copy).
  • hero — text-base (1.0625rem) / leading-[1.6] / text-muted. Used for hero subtitles.
  • card — text-card-copy (0.8125rem) / leading-[1.55] / text-muted. Sized for card descriptions; not yet wired into PostCard, which renders its own excerpt markup instead.

Templates / routes

components/templates/home-page-template.tsx, post-page-template.tsx. Route files pass data and choose whether optional image props exist.

Route Template placement Optional image rule
app/page.tsx SiteHeader → HomeHero → LatestPosts → SiteFooter Pass heroImage only when available. HomeHero adds has-media internally.
app/writing/[slug]/page.tsx SiteHeader → ArticleHeader → ArticleBody Pass coverImage only when available. ArticleHeader removes the figure when missing.
Implementation rule: atoms/molecules should not add route margins like mt-section or page max-widths. Use templates and page files for placement. Organisms may own internal section layout, but not overall page sequence.
